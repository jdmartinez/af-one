<task type="auto">
  <name>task 1: Fix period picker reload</name>
  <files>AFOne/Features/Timeline/TimelineView.swift</files>
  <action>
In TimelineView, add `.onChange(of: viewModel.selectedPeriod)` to reload data when period changes. The current Picker updates selectedPeriod but doesn't trigger data reload.

Add this modifier to the Picker in timelineContent:
```swift
.onChange(of: viewModel.selectedPeriod) { _, newPeriod in
    Task {
        await viewModel.loadData()
    }
}
```
  </action>
  <verify>
Switch between 7 Days and 30 Days - data should refresh and show different number of days
  </verify>
  <done>Picker changes trigger data reload</done>
</task>

<task type="auto">
  <name>task 2: Fix empty state detection</name>
  <files>AFOne/Features/Timeline/TimelineView.swift</files>
  <action>
The empty state check uses `viewModel.days.isEmpty` but days is never empty - it always gets populated with fallback data. Change the check to detect if all days have `hasData == false`:

Replace line 145:
```swift
} else if viewModel.days.isEmpty {
```

With:
```swift
} else if viewModel.days.allSatisfy({ !$0.hasData }) {
```

This will show the empty state when there are no real data points.
  </action>
  <verify>
Fresh install with no HealthKit data shows empty state message
  </verify>
  <done>Empty state displays when no health data</done>
</task>

<task type="auto">

  <files>AFOne/Features/Timeline/TimelineView.swift</files>
  <action>
In `selectDay()` function, when the selected day has no data (`hasData == false`), the hourlyData should show all `.unknown` state, not default to `.normal`.

Fix line 108:
```swift
let rhythm: RhythmState = hourEpisodes.isEmpty ? .normal : .af
```

Change to check if the day has data first:
```swift
let rhythm: RhythmState
if !day.hasData {
    rhythm = .unknown
} else {
    rhythm = hourEpisodes.isEmpty ? .normal : .af
}
```
  </action>
  <verify>
